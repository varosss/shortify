# Shortify

Лёгкий и производительный сервис коротких ссылок, написанный на Go.  
Проект демонстрирует построение REST API с использованием принципов Clean Architecture, разделения ответственности и масштабируемой структуры кода.

---

## Возможности

- Создание короткой ссылки из длинного URL  
- HTTP-редирект по короткому коду  
- Валидация входных данных  
- Чистая архитектура (handler → usecase → repository → domain)  
- Graceful shutdown сервера  
- Готовность к расширению (БД, кэш, статистика)

---

## Архитектура проекта

Проект построен по принципам Clean Architecture:

```
internal/
├── domain/ # Бизнес-сущности и интерфейсы
├── usecase/ # Бизнес-логика
├── repository/ # Работа с хранилищем
└── transport/ # HTTP-слой

cmd/
└── main.go # Точка входа
```

### Поток обработки запроса

```
HTTP → Handler → UseCase → Repository → Storage
```

Такое разделение позволяет:
- упростить тестирование
- снизить связанность компонентов
- легко заменить хранилище (in-memory → PostgreSQL → Redis)
- масштабировать проект без изменения бизнес-логики

## Стек технологий

- Go 1.22+
- net/http (или Gin — в зависимости от реализации)
- UUID / Base62 для генерации короткого кода
- Docker (опционально)

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/varosss/shortify.git
cd shortify
```

### 2. Собрать образ

```bash
make build.prod
```

### 3. Запуск контейнеров

```bash
docker compose up
```

### 4. Запуск миграций

```bash
make docker-migrate-up
```

## API

### Создание короткой ссылки

**POST** /api/v1/shorten

**Request**

```json
{
  "url": "https://example.com/very/long/url"
}
```

**Response**
```json
{
  "short_url": "http://localhost:81/abc123"
}
```

### Редирект

**GET** /{code}

При переходе по короткому коду сервер возвращает HTTP 302 и перенаправляет пользователя на оригинальный URL.
